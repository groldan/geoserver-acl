== Level 1: System Context Whitebox

=== System Decomposition

GeoServer ACL consists of two main components that work together to provide authorization services for GeoServer:

[plantuml, system-whitebox, svg]
....
@startuml
!include https://raw.githubusercontent.com/plantuml-stdlib/C4-PlantUML/master/C4_Container.puml

Person(admin, "GeoServer Admin", "Defines access rules")
Person(user, "End User", "Accesses geospatial data")

System_Boundary(aclSystem, "GeoServer ACL") {
    Container(aclService, "ACL Service", "Spring Boot", "Rule management and authorization decisions")
    Container(geoserverPlugin, "GeoServer ACL Plugin", "Java", "Enforces authorization in GeoServer")
}

System_Ext(geoserver, "GeoServer", "OGC-compliant geospatial server")

Rel(admin, aclService, "Manages rules", "REST API")
Rel(user, geoserver, "Requests data", "OGC Services")
Rel(geoserver, geoserverPlugin, "Uses")
Rel(geoserverPlugin, aclService, "Requests authorization", "REST API")

@enduml
....

=== Components

[cols="1,4,4a"]
|===
|Component |Responsibility |Interfaces

|ACL Service
|Standalone application that manages access rules and provides authorization decisions
|* REST API for rule management
* REST API for authorization decisions
* Database interface for persistence

|GeoServer ACL Plugin
|GeoServer extension that integrates with GeoServer's security subsystem and enforces access rules
|* GeoServer ResourceAccessManager implementation
* OWS Service request filters
* Administrative UI extensions
|===

== Important Interfaces

=== Domain Service Interfaces

The domain service interfaces define the core functionality of the ACL system.

[source,java]
----
public interface RuleAdminService {
    /** Rule management operations */
    Rule get(String id);
    Rule insert(Rule rule);
    Rule update(Rule rule);
    boolean delete(String id);
    
    /** Rule querying operations */
    Stream<Rule> getAll();
    Stream<Rule> getAll(RuleFilter filter, Integer page, Integer size);
    long count(RuleFilter filter);
    /** ... */
}

public interface AuthorizationService {
    /** Authorization decision operations */
    AccessInfo getAccessInfo(AccessRequest request);
    AccessInfo getWorkspaceAccessInfo(WorkspaceAccessRequest request);
    AdminAccessInfo getAdminAuthorization(AdminAccessRequest request);
    /** ... */
}
----

=== REST API Interfaces

The REST API is defined using OpenAPI and exposes endpoints for rule management and authorization.

[source,yaml]
----
paths:
  /rules:
    get:
      summary: List all rules
      # ...
    post:
      summary: Create a new rule
      # ...
  /rules/{id}:
    get:
      summary: Get a rule by ID
      # ...
    put:
      summary: Update a rule
      # ...
    delete:
      summary: Delete a rule
      # ...
  /authorize:
    post:
      summary: Get authorization decision
      # ...
----

=== GeoServer Plugin Interfaces

The GeoServer plugin implements GeoServer's security interfaces to enforce access control.

[source,java]
----
public class ACLResourceAccessManager implements ResourceAccessManager {
    @Override
    public WorkspaceAccessLimits getAccessLimits(Authentication user, WorkspaceInfo workspace) {
        // Request workspace access decision from ACL service
    }
    
    @Override
    public LayerGroupAccessLimits getAccessLimits(Authentication user, LayerGroupInfo layerGroup) {
        // Request layer group access decision from ACL service
    }
    
    @Override
    public DataAccessLimits getAccessLimits(Authentication user, LayerInfo layer) {
        // Request layer access decision from ACL service
    }
    
    // Other ResourceAccessManager methods...
}
----
