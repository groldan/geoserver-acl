== Business Context

GeoServer ACL operates within the context of Geographic Information System (GIS) data infrastructure, particularly in environments where controlling access to geospatial data is critical.

=== Business Context Diagram

[plantuml, business-context, svg]
....
@startuml
!include https://raw.githubusercontent.com/plantuml-stdlib/C4-PlantUML/master/C4_Context.puml

Person(admin, "GeoServer Admin", "Configures access rules and GeoServer")
Person(user, "End User", "Accesses geospatial data via OGC services")

System(geoserver, "GeoServer", "OGC-compliant geospatial data server")
System(aclSystem, "GeoServer ACL", "Authorization system that enforces access rules")
System_Ext(authSystem, "Authentication System", "Verifies user identity (LDAP, OAuth2, etc.)")
System_Ext(spatialDB, "Spatial Database", "Stores geospatial data (PostGIS, etc.)")

Rel(user, geoserver, "Requests geospatial data from", "OGC services (WMS, WFS, etc.)")
Rel(admin, geoserver, "Configures")
Rel(admin, aclSystem, "Manages access rules")
Rel(geoserver, aclSystem, "Requests authorization decisions from")
Rel(geoserver, authSystem, "Authenticates users via")
Rel(geoserver, spatialDB, "Retrieves data from")
Rel(aclSystem, authSystem, "Uses identity information from")

@enduml
....

=== External Systems and Interfaces

[cols="1,4,2,4"]
|===
|System |Description |Interface Type |Data Exchanged

|GeoServer
|OGC-compliant geospatial data server
|REST API, Java API
|Access request details, resource information, authorization decisions

|Authentication Systems
|External identity providers (LDAP, OAuth2, etc.)
|Authentication tokens
|User identity and role information

|Spatial Database
|Store for geospatial data (PostGIS, Oracle Spatial, etc.)
|SQL, JDBC
|Spatial data for filtering

|Management Clients
|Web browsers, API clients
|HTTP/REST
|Rule management commands
|===

=== Technical Interfaces

[cols="1,4,2,4a"]
|===
|Interface |Description |Protocol |Details

|ACL Service REST API
|API for rule management and authorization decisions
|HTTP/REST
|* Endpoints:
** Rule management (CRUD)
** Authorization requests
** Administrative operations
* Authentication: 
** Basic auth
** JWT/OAuth2 tokens
* Data format: JSON

|GeoServer Plugin Interface
|Integration with GeoServer security framework
|Java API
|* Security filter integration
* Catalog listener for resource events
* Request interceptor for OGC services
* Implements ResourceAccessManager interface

|Database Connection
|Storage for rule definitions
|JDBC
|* JPA entities for rules
* Transaction management
* Connection pooling
* Schema migration via Flyway
|===

=== Domain Interfaces

[cols="1,4,4a"]
|===
|Domain Interface |Purpose |Key Operations

|Rule Management
|Manage the definition and storage of access rules
|* Create, read, update, delete rules
* Search and filter rules
* Import/export rule sets
* Validate rule consistency

|Authorization
|Determine if access to a resource is permitted
|* Check if a user can access a resource
* Determine spatial limitations
* Determine attribute limitations
* Apply rule priorities
* Cache authorization decisions

|GeoServer Integration
|Connect authorization decisions to GeoServer
|* Intercept OGC service requests
* Filter catalog resources
* Apply data access restrictions
* Apply administrative access restrictions
|===
